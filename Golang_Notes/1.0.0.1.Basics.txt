Golang (also known as Go) is an cross platform, open-source programming language developed by Google. 

    Why Go?
        * It is designed for simplicity, efficiency, and strong support for concurrent programming. 
        * Go can be used to create high-performance applications; as Go has fast run time and compilation time, making it suitable for building scalable and efficient software.
        * Go is statically typed, compiled, and known for its performance and ease of use in building scalable and maintainable software.
        * Go has a clean syntax, built-in garbage collection, and a powerful standard library that makes it suitable for a wide range of applications, from web development to system programming.

    Go language was designed to address common issues in software development, such as slow compilation times, dependency management and the complexity of concurrent programming - addressed through Go's built-in support for goroutines and channels. 

History of Go:-
    Go was created at Google in 2007 by Robert Griesemer, Rob Pike, and Ken Thompson. 
    The language was designed to address the challenges of software development at Google, such as slow compilation times and the need for efficient concurrency. 
    Go was publicly announced in November 2009 and has since gained popularity for its simplicity, performance, and ease of use. 
    Go has been used in various projects at Google and has also been adopted by many other companies and developers worldwide. 

Go Use Cases:-
    Go is used in a wide range of applications in various industries, including:
    - Web development: Go is popular for building web servers and APIs due to its performance and simplicity.
    - Networking: Go's built-in support for concurrency makes it ideal for building networked applications, such as chat servers and proxies.
    - Command-line tools: Go's simplicity and performance make it a great choice for building command-line tools and utilities.
    - Cloud native: Go is widely used in cloud infrastructure and services, such as Kubernetes and Docker.
    - Microservices: Go's support for concurrency makes it a great choice for building microservices architectures.
    - System programming: Go can be used for system-level programming tasks, such as writing command-line tools and network servers.
    - Data processing: Go's performance and efficiency make it suitable for data processing tasks, such as ETL pipelines and data analysis.

0. Setting Up the Go Environment
    To set up the Go environment on your machine, follow these steps:
    1. Download and Install Go: Visit the official Go website (https://golang.org/dl/) and download the appropriate installer for your operating system. Follow the installation instructions to complete the setup.
    2. Set Up Go Workspace: Go uses a workspace to organize your code. Create a directory for your Go projects (e.g., ~/go) and set the GOPATH environment variable to point to this directory. 
    3. Verify Installation: Open a terminal and run the command go version to verify that Go is installed correctly. You should see the installed version of Go displayed.
    4. Create a Go Project: Inside your workspace, create a new directory for your project (e.g., ~/go/src/myproject) and navigate to that directory in the terminal.
    5. Write Your First Go Program: Create a new file named main.go in your project directory and add the "Hello, World!" program or any other Go code you want to run.
    6. Run Your Go Program: In the terminal, navigate to your project directory and run the command go run main.go to execute your Go program. You should see the output in the terminal.

1. Go Toolchain and Common Commands
    Go toolchain overview:- 
    The Go toolchain provides a set of commands for building, running, testing, and managing Go programs, modules, packages, and dependencies. Here are some common commands you should know:
    // Onefile management and running
        - "go run <filename.go>" to compile and run a Go program in one step
        - "go build <filename.go>" to compile a Go program into an executable file
        - "go install <filename.go>" to compile and install a Go program in your Go workspace
        - "go fmt <filename.go>" to format your Go code according to the standard style guidelines
        - "go vet <filename.go>" to analyze your Go code for potential errors and issues
        - "go test <filename.go>" to run tests defined in your Go program

    // Module management and building
        - "go mod init <module_name>" to initialize a new Go module with the specified name, creating a go.mod file in the current directory
        - "go build ." to compile the Go program in the current directory, which is useful when you have multiple Go files in the same directory and want to build them together as a module
        - "go run ." to run the program within the module context. This is useful when you have multiple Go files in the same directory and want to run them together as a module.
        - "go clean" to remove compiled files and clean up the workspace
        - "go test" to run tests in your Go project
        
    // Dependency management
        - "go get <package>" to download and install a Go package and its dependencies
        - "go list <package>" to list the packages in your module or workspace
        - "go list -m all" to list all the modules and their versions in your project

    // Additional module management commands
        - "go mod tidy" to clean up the go.mod file and remove any unused dependencies
        - "go mod vendor" to create a vendor directory and copy all the dependencies into it, which can be useful for offline builds or when you want to ensure that specific versions of dependencies are used
        - "go mod graph" to display the dependency graph of your Go modules
        - "go mod why <package>" to explain why a specific package is needed in your project
        - "go mod verify" to verify that the dependencies in your go.mod file match the expected checksums in the go.sum file
        - "go mod edit -require <module>@<version>" to add a specific module and version to your go.mod file
        - "go mod edit -replace <old_module>=<new_module>" to replace a module with another module in your go.mod file, which can be useful for testing or using a local version of a dependency
        - "go mod edit -dropreplace <module>" to remove a replace directive from your go.mod file
        
    // Additional Go commands
        - "go version" to check the installed version of Go
        - "go env" to display the Go environment variables and settings
        - "go help" to get help on Go commands and usage
        - "go doc <package>" to view documentation for a Go package

3. Go Program Structure and Syntax
        Syntax of a Go Program
            package <package_name>                          // Package declaration, where <package_name> is the name of the package being defined. The main package is used for executable programs.

            import <package_name>
                    <github.com/username/repository>        // Importing a package from GitHub sourcecode
            
            func <function_name>(parameters) return_type {  // Function declaration, where <function_name> is the name of the function, parameters are the input parameters (if any), and return_type is the type of value returned by the function (if any).
                // function body
            }

        A Go program typically consists of the following components:
        - Package declaration: Every Go file starts with a package declaration. The main package is used for executable programs.
        - Import statements: You can import other packages to use their functionality. The fmt package is commonly used for input and output operations.
        - Functions: Functions are defined using the func keyword. The main function is the starting point of the program.

        What is a package in Go?
        A package in Go is a collection of related Go source files that are compiled together. 
        It serves as a way to organize and reuse code. 
        Each package has a unique name, and you can import packages to use their functionality in your program. 
        The main package is a special package that defines the entry point of an executable program.

        What is a module in Go?
        A module in Go is a collection of related Go packages that are versioned together. 
        It provides a way to manage dependencies and versioning for your Go projects. 
        A module is defined by a go.mod file, which specifies the module's name and its dependencies. 
        Modules allow you to easily manage and share code across different projects.

    Key Components:
        * go.mod file: This file defines the module and its dependencies. It is created when you run "go mod init" and is used to manage the versions of the packages your project depends on.
        * main.go file: This is the main source file for your Go program. It contains the main function, which is the entry point of the program. You can have multiple source files in your project, but the main.go file is typically where you write the core logic of your application.
        * go.sum file: This file is automatically generated when you run "go mod tidy" or "go get". It contains the checksums of the dependencies used in your project, ensuring that the correct versions of the packages are used.
        * vendor directory: This directory can be used to store copies of the dependencies your project relies on. It is created when you run "go mod vendor" and allows you to include the dependencies directly in your project, which can be useful for offline builds or when you want to ensure that specific versions of dependencies are used.
        * bin directory: This directory is where the compiled executable files are stored when you build your Go program. When you run "go build", the resulting binary is placed in this directory, allowing you to easily execute your program.
        * pkg directory: This directory is used to store compiled package files. When you build your Go program, the compiled packages are stored in this directory, allowing for faster builds and easier management of dependencies.
        * src directory: This directory is where you typically place your Go source files. It is the main location for your code, and you can organize it into subdirectories based on the structure of your project.
        * test directory: This directory is commonly used to store test files for your Go program. It is a convention to place test files in a separate directory to keep them organized and separate from the main source code.
        * README.md file: This file is used to provide documentation and information about your Go project. It typically includes an overview of the project, instructions for installation and usage, and any other relevant information for users or contributors.
        * LICENSE file: This file is used to specify the license under which your Go project is distributed. It is important to include a license to clarify the terms of use and distribution for your code.
        * .git directory: This directory is created when you initialize a Git repository for your Go project. It contains all the version control information and history for your project, allowing you to track changes and collaborate with others.

6. Variables and Constants
        * In Go, you can declare variables using 
            * the var keyword 
            * or use short variable declaration with :=. 
        * Go is a statically typed language, which means that the type of a variable must be specified at the time of declaration. However, Go also supports type inference, allowing you to omit the type when it can be determined from the context.
        * Variable can be initialized with a value at the time of declaration, or it can be declared without an initial value.
        * If not initialized, it will be assigned the default value for its type (e.g., 0 for int, "" for string, false for bool).
        * Variables can be assigned new values after declaration at a later point, while constants cannot be changed once they are set.  
        * Constants can be declared using the const keyword and must be assigned a value at the time of declaration. Once a constant is set, its value cannot be changed throughout the program. Constants are typically used for values that should remain unchanged, such as mathematical constants or configuration values.

            Syntax for variable declaration and initialization:
                var <variable_name> <type>        //Declaration (without initial value)
                var <variable_name> <type> = <value>    // Declare with an initial value, where the type is explicitly specified
                <variable_name> := <value>          // Short variable declaration with type inference, where the type is determined from the value assigned
                var <variable_name> = <value>     // Declare a variable with an initial value, where the type is inferred from the value assigned
                const <constant_name> = <value>     // untyped constant declaration, where the type is inferred from the value assigned
                const <constant_name> <type> = <value> // typed constant declaration

            ```go
            var name string = "Go"      // Declare a variable of type string and initialize it with the value "Go"
            age := 10                   // Short variable declaration, Go infers the type as int
            const pi = 3.14             // Declare a constant of type float64 and assign it the value 3.14. Type inference is used to determine the type of pi as float64.        
            ```

        * Difference Between var and :=
            - var is used for declaring variables with an explicit type, while := is a shorthand for declaring and initializing variables without specifying the type (type inference).
            - variable declarion and value assignment must be done in a single step using :=, while var allows for separate declaration and assignment.
            - var can be used to declare variables at the package level(can be used inside and outside functions), while := can only be used within functions.

        * Multiple variable declaration and assignment
            ```go
            var a, b, c int = 1, 2, 3   // Declare multiple variables of type int and initialize them with values
            x, y := 4, 5                 // Short variable declaration for multiple variables with type inference
            ```

        * Variable declaration in a Block - Multiple variable declarations can also be grouped together into a block for greater readability:
            ```go
            var (
                name string = "Go"
                age  int    = 10
            )
            ```
        
        * Constants can also be declared in a block:
            ```go
            const (
                pi    = 3.14
                e     = 2.718
                phi   = 1.618
            )
            ```

        * iota package provides a way to generate a sequence of related constants using the iota identifier. It is commonly used for creating enumerated constants.
            ```go
            const (
                Sunday = iota
                Monday
                Tuesday
                Wednesday
                Thursday
                Friday
                Saturday
            )
            ```

        In Go, iota is a predeclared identifier used within const blocks to simplify the definition of incrementing, auto-generated, and often related integer constants. 
            Key features of iota include:
                * Auto-incrementing: iota represents successive untyped integer constants.
                * Reset behavior: iota resets to 0 at the start of each const declaration block.
                * Expressions: It can be used in expressions, such as 1 << iota for bit shifting to create powers of two.
                * Skipping values: The blank identifier _ can be used to skip values in a sequence.
                * Enumerations: iota is commonly used to define enumerated types (enums) in Go.
                * Implicit assignment: The expression A = iota can be shortened to A

            Example usage of iota:
                package main
                import "fmt"

                func main() {
                    const (                     // Basic iota usage
                        A = iota                // 0
                        B = iota                // 1
                        C = iota                // 2
                    )
                    fmt.Println(A, B, C)        // Output: 0 1 2

                    // Expression with iota
                    const (
                        _ = iota                // Skip 0
                        KB = 1 << (10 * iota)   // 1 << 10
                        MB = 1 << (10 * iota)   // 1 << 20
                    )
                    fmt.Println(KB, MB)         // Output: 1024 1048576
                }

        * Variable Shadowing
            Variable shadowing occurs when a variable declared within a certain scope (e.g., inside a function or block) has the same name as a variable declared in an outer scope. In such cases, the inner variable "shadows" the outer variable, meaning that within the inner scope, references to that variable name will refer to the inner variable rather than the outer one. This can lead to confusion and bugs if not handled carefully, as it may not be immediately clear which variable is being referenced in different parts of the code.

            Example of variable shadowing:
                package main
                import "fmt"

                var x = 10 // Outer variable

                func main() {
                    fmt.Println("Outer x:", x) // Output: Outer x: 10

                    x := 20 // Inner variable shadows the outer variable
                    fmt.Println("Inner x:", x) // Output: Inner x: 20

                    {
                        x := 30 // Another inner variable shadows the previous inner variable
                        fmt.Println("Innermost x:", x) // Output: Innermost x: 30
                    }

                    fmt.Println("Inner x after block:", x) // Output: Inner x after block: 20
                }

        * Variable Scope
            In Go, the scope of a variable is determined by where it is declared. Variables declared inside a function are local to that function and cannot be accessed outside of it. Variables declared at the package level (outside of any function) are global and can be accessed from any function within the same package. Additionally, variables declared within a block (e.g., inside an if statement or for loop) are only accessible within that block.

        * Variable Naming Conventions 
            In Go, variable names should be descriptive and follow certain conventions. 
                * should start with a letter (or an underscore).
                * can contain only letters, digits, and underscores. 
                * cannot start with a digit and cannot contain spaces or special characters.
                * should not be a reserved keyword in Go (e.g., var, func, if, else, etc.).
                * should be unique within its scope to avoid naming conflicts and ensure clarity in the code.
                * case-sensitive, meaning that myVariable and MyVariable would be considered different variables.
                * should not be too long or too short; it should strike a balance between being descriptive and concise. while there is no strict limit on the length.
                * It is common practice to use camelCase for variable names (e.g., myVariable) and to avoid using underscores or hyphens.
                * should be concise yet meaningful, providing clarity about the purpose of the variable in the context of the code.
                * Constants should be named using uppercase letters with underscores to separate words (e.g., MAX_VALUE) to distinguish them from regular variables.

7. Line Breaks and Indentation
    Go uses line breaks and indentation to define the structure of the code. 
    Each statement should be on its own line, and blocks of code (such as function bodies, loops, and conditionals) should be indented using tabs or spaces (Go convention is to use tabs). 
    Proper indentation helps improve the readability of the code and makes it easier to understand the flow of the program.
        ```go
        func main() {
            fmt.Println("Hello, World!") // Properly indented code block
        }
        ```

    Line breaks are used to separate statements and improve readability. In Go, a line break is not allowed in the middle of a statement or expression. If you need to break a long statement into multiple lines for readability, you can use parentheses to group the expression and allow for line breaks.

        Example of line breaks:
        ```go
        result := add(5, 3) + add(2, 4) // This is valid

        result := (add(5, 3) +            // This is also valid with parentheses ()
                    add(2, 4))

        result := add(5, 3) +            // This is invalid and will cause a syntax error
        add(2, 4)
        ```

    Line break and continuation rules:
        - A line break is not allowed in the middle of a statement or expression unless it is within parentheses, brackets, or braces.
        - If a line ends with an operator (e.g., +, -, *, /), the next line is considered a continuation of the previous line.
        - If a line ends with a comma, the next line is also considered a continuation of the previous line.

    Line Break restrictions in Go

8. Output function in Go
    The fmt package is commonly used for formatted I/O operations in Go. 
    It provides various functions for printing output to the console. 
    Some of the commonly used functions include:
        - fmt.Print(): This function prints the provided arguments with their default format. 
        It is useful when you want to print multiple items on the same line.
            ```go
            fmt.Print("Hello, ") // Output: Hello, 
            fmt.Print("World!")  // Output: World!
            ```
            
        - fmt.Println(): This function prints the provided arguments followed by a newline character and whitespace is added between the arguments. It is used for simple output without any formatting.
            ```go
            fmt.Println("Hello, World!") // Output: Hello, World!
            ```

        - fmt.Printf(): This function allows you to format the output using format specifiers. It is useful when you want to include variables or control the formatting of the output.
            ```go
            name := "Go"
            age := 10
            fmt.Printf("My name is %s and I am %d years old.\n", name, age) // Output: My name is Go and I am 10 years old.
            ```

        - fmt.Sprintf(): This function is similar to fmt.Printf(), but instead of printing the output to the console, it returns the formatted string. It can be useful when you want to store the formatted output in a variable or use it in other parts of your code.
            ```go
            name := "Go"
            age := 10
            message := fmt.Sprintf("My name is %s and I am %d years old.", name, age)
            fmt.Println(message) // Output: My name is Go and I am 10 years old.
            ```

9. Control Flow in Go
    Go supports various control structures, such as if statements, for loops, and switch statements. These structures allow you to control the flow of your program based on conditions and iterations.

    If statements are used for conditional execution:-
        ```go
        if age > 18 {
            fmt.Println("You are an adult.")
        } else {
            fmt.Println("You are a minor.")
        }
        ```

    for loops are used for iteration:
        ```go
        for i := 0; i < 5; i++ {
            fmt.Println(i)
        }
        ```

    for as a while loop
        ```go
        i := 0
        for i < 5 {
            fmt.Println(i)
            i++
        }
        ```

    for as an infinite loop
        ```go
        for {
            fmt.Println("This will run forever.")
        }
        ```

    for range loops are used to iterate over collections:
        ```go
        numbers := []int{1, 2, 3, 4, 5}
        for index, value := range numbers {
            fmt.Printf("Index: %d, Value: %d\n", index, value)
        }
        ```

    Break, Continue, and goto statements are used for controlling the flow of loops and blocks of code:
        ```go
        for i := 0; i < 5; i++ {
            if i == 2 {
                continue // Skip the rest of the loop body when i is 2
            }
            if i == 4 {
                break // Exit the loop when i is 4
            }
            fmt.Println(i)
        }
    
        	// for loop as a while loop
                j := 0
                for j < 5 {
                            if j == 3 {
                        goto skip // Jump to the label skip when j is 3
                    }
                    fmt.Println("While loop iteration:", j)
                    j++
                }
            skip:
                fmt.Println("Exited the loop.")

    return statements are used to exit from a function and optionally return one or more values:
        ```go
        func add(a int, b int) int {
            return a + b // Return the sum of a and b
        }

        // another example of return statement
        func divide(a int, b int) (int, error) {
            if b == 0 {
                return 0, fmt.Errorf("cannot divide by zero") // Return an error if b is zero
            }
            return a / b, nil               // Return the result of a divided by b and nil error
        }
        ```


    switch statements are used for multi-way branching based on the value of an expression:
        ```go
        switch day := "Monday"; day {
        case "Monday":
            fmt.Println("It's the start of the week.")
        case "Friday":
            fmt.Println("It's the end of the week.")
        default:
            fmt.Println("It's a regular day.")
        }
        ```

    Select-case statements are used for handling multiple channel operations in concurrent programming:
        ```go
        select {
        case msg := <-channel1:
            fmt.Println("Received message from channel1:", msg)
        case msg := <-channel2:
            fmt.Println("Received message from channel2:", msg)
        default:
            fmt.Println("No messages received.")
        }
        ```

        type-switch statements are used for type assertions and type-based branching:
        ```go

            fmt.Println("Value is a string:", v)
        case int:
            fmt.Println("Value is an int:", v)
        default:
            fmt.Printf("Value is of type %T\n", v)
        }
        ```

        Other Control Flow Statements:
            - defer: Used to schedule a function call to be executed after the surrounding function returns. It is often used for cleanup tasks, such as closing files or releasing resources, ensuring that they are executed regardless of how the function exits (e.g., normal return, panic, etc.). and multiple defer statements execute in Last-In, First-Out (LIFO) order.

                Example of defer:
                ```go
                func main() {
                    defer fmt.Println("This will be printed last.")
                    fmt.Println("This will be printed first.")
                }
                ```

            - panic: Used to cause a runtime error and stop the normal execution of the program. It is typically used to indicate an unrecoverable error or exceptional condition. When a panic occurs, the program will print a stack trace and exit with a non-zero status code.

                    Example of panic:
                    ```go
                    func main() {
                        panic("Something went wrong!") // This will cause a panic and stop the program
                    }
                    ```

            - recover: Used to regain control of a panicking goroutine. It allows you to handle the panic and prevent the program from crashing and resume normal program execution. The recover function can only be called within a deferred function, and it will return the value passed to panic if a panic is occurring, or nil if there is no panic.

                    Example of recover:
                    ```go
                    func main() {
                        defer func() {
                            if r := recover(); r != nil {
                                fmt.Println("Recovered from panic:", r) // This will handle the panic and print the message
                            }
                        }()
                        panic("Something went wrong!") // This will cause a panic, but it will be recovered by the deferred function
                    }
                    ```

10. Operators in Go
    Go supports a variety of operators for performing operations on variables and values. These include:
        - Arithmetic operators: +, -, *, /, % (modulo), ++ (increment), -- (decrement)
        - Assignment operators: =, +=, -=, *=, /=, %=, &=, |=, ^=, <<=, >>=
        - Comparison operators: ==, !=, <, >, <=, >=
        - Logical operators: && (logical AND), || (logical OR), ! (logical NOT)
        - Bitwise operators: &, |, ^ (XOR), &^ (AND NOT), << (left shift), >> (right shift)
        - Other operators: & (address of), * (pointer dereference), <- (channel receive)

    Key difference between Go and other languages:
        * No prefix operators: In Go, there are no prefix increment (e.g., ++a) or decrement (e.g., --a) operators. Instead, Go only supports postfix increment (a++) and decrement (a--) operators. This means that you can only use the increment or decrement operator after the variable, and it will increment or decrement the variable's value after its current value is used in an expression.
        * Statements, Not Expressions: In Go, a++ and a-- are considered statements on their own line, not expressions that return a value. This design choice prevents common, confusing bugs related to the order of evaluation within complex expressions, such as x = x++ + x.
        * += 1 Alternative: In Go, instead of using a++ to increment a variable, you can use the += operator to achieve the same result. For example, a += 1 is equivalent to a++ and will increment the value of a by 1.
        * No Operator Overloading: Go does not support user-defined operator overloading, which simplifies the language and keeps behavior consistent across all code.
        * No Pointer Arithmetic (General): While Go has pointers, it does not support general pointer arithmetic (e.g., incrementing a pointer to move to the next memory address)
        * Go's design emphasizes simplicity and clarity.

        * The ++a syntax (pre-increment) does not exist in the Go programming language. Go only supports the postfix increment operator (a++) and does not have a pre-increment operator. In Go, you can only use the increment operator after the variable (postfix), and it will increment the variable's value after its current value is used in an expression.

        Example of using operators:
        ```go
        // Arithmetic operators
        a := 10
        b := 5
        sum := a + b           // Addition
        difference := a - b    // Subtraction
        product := a * b       // Multiplication
        quotient := a / b      // Division
        remainder := a % b     // Modulo
        a++                    // Increment a by 1 (postfix)
        b--                    // Decrement b by 1 (postfix)
        // Assignment operators
        a = 10                 // Assign 10 to a
        a += 5                 // Equivalent to a = a + 5
        b *= 2                 // Equivalent to b = b * 2   
        a -= 3                 // Equivalent to a = a - 3
        b /= 2                 // Equivalent to b = b / 2
        a %= 4                 // Equivalent to a = a % 4
        a &= 3                 // Equivalent to a = a & 3
        b |= 1                 // Equivalent to b = b | 1
        a ^= 2                 // Equivalent to a = a ^ 2
        a <<= 1                // Equivalent to a = a << 1
        b >>= 1                // Equivalent to b = b >> 1
        // Comparison operators
        isEqual := (a == b)   // Check if a is equal to b
        isNotEqual := (a != b) // Check if a is not equal to b
        isLess := (a < b)     // Check if a is less than b
        isGreater := (a > b)  // Check if a is greater than b
        isLessOrEqual := (a <= b) // Check if a is less than or equal to b
        isGreaterOrEqual := (a >= b) // Check if a is greater than or equal to b
        // Logical operators
        isTrue := (a > 0) && (b > 0) // Logical AND
        isEitherTrue := (a > 0) || (b > 0) // Logical OR
        isNotTrue := !(a > 0) // Logical NOT 
        // Bitwise operators
        bitwiseAnd := a & b   // Bitwise AND
        bitwiseOr := a | b    // Bitwise OR
        bitwiseXor := a ^ b   // Bitwise XOR
        bitwiseAndNot := a &^ b // Bitwise AND NOT
        leftShift := a << 1   // Left shift
        rightShift := a >> 1  // Right shift
        // Other operators
        addressOfA := &a      // Get the memory address of a
        pointerToA := *addressOfA // Dereference the pointer to get the value of a
        // channel receive operator is used in concurrent programming with channels
        	// Example of channel receive operator
        channel := make(chan int)
        go func() {

        }()
        value := <-channel // Receive a value from the channel
        fmt.Println(value) // Print the received value      
        ```

    * Operator Precedence and Associativity
        In Go, operator precedence determines the order in which operators are evaluated in an expression. Operators with higher precedence are evaluated before operators with lower precedence. If two operators have the same precedence, their associativity determines the order of evaluation. 

        The operator precedence in Go, from highest to lowest, is as follows:
            1. Parentheses: ( )
            2. Unary operators: +, -, !, ^, &, * (address of), <- (channel receive)
            3. Multiplicative operators: *, /, %
            4. Additive operators: +, -
            5. Bitwise shift operators: <<, >>
            6. Relational operators: <, <=, >, >=
            7. Equality operators: ==, !=
            8. Bitwise AND: &
            9. Bitwise XOR: ^
            10. Bitwise OR: |
            11. Logical AND: &&
            12. Logical OR: ||
            13. Assignment operators: =, +=, -=, *=, /=, %=, &=, |=, ^=, <<=, >>=

            * & + - ! ^	            Right to Left (Unary)
            * / % << >> & &^	    Left to Right (Multiplicative, Shift, Bitwise AND, Bitwise Clear)
            + - `	                   ^`

Key Concepts
    * Precedence: An expression like 7 + 3 * 2 results in 13, not 20, because multiplication (*) has a higher precedence than addition (+).
    * Associativity: For operators of the same precedence, associativity rules apply. For example, 9 / 3 * 3 is evaluated as (9 / 3) * 3 (left-to-right) resulting in 9, not 9 / (3 * 3).
    * Overriding Order: Parentheses (()) can force a specific order of operations. (7 + 3) * 2 evaluates the addition first, resulting in 20.
    * Order of Evaluation vs. Associativity: Associativity determines how operands are grouped, but it does not specify the order in which the operands themselves are evaluated. For example, in the expression f1() * f2(), while the grouping is clear, the Go compiler might call f2() before f1() for optimization purposes unless the language specification guarantees an order.
