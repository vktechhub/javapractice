Level 1 — Go Foundations (Beginner, 4–6 weeks)

Goal: Build strong Go basics and write clean, testable code.
Go setup, modules, workspace layout, go mod, package design, naming conventions.
Core language: types, structs, interfaces, pointers, slices/maps, methods, errors, panic/recover.
Control flow and idiomatic coding patterns (if, for, switch, defer).
Concurrency basics: goroutines, channels, select, sync primitives, race detector.
Clean code practices: layering, SOLID in Go style, linting, formatting, unit tests, table-driven tests.
Mini project: CLI + small HTTP service with config, logging, tests.
Level 2 — Backend Engineering with Go (Intermediate, 6–8 weeks)

Goal: Build production-style services with data + messaging integration.
REST API design (versioning, pagination, filtering, idempotency, error contracts).
API consumer handling: JSON/XML, content negotiation, retries, timeouts, circuit breakers, SDK/client patterns.
Microservices fundamentals: boundaries, service discovery, config, observability, resilience patterns.
RDBMS with Go (database/sql, transactions, indexing, connection pooling, migrations).
NoSQL with Go (MongoDB/Redis patterns, document/key-value modeling, consistency trade-offs).
Kafka with Go: producer/consumer groups, partitioning, schema strategy, at-least-once vs exactly-once patterns.
Mini project: 2-service system (Order + Inventory) using REST + DB + Kafka events.
Level 3 — Advanced Go for Cloud-Native Systems (Advanced, 8–10 weeks)

Goal: Optimize, secure, and operate high-performance Go systems in cloud environments.
Performance tuning: profiling (pprof), memory/CPU optimization, GC awareness, benchmarking (go test -bench).
Compiler/build options: go build flags, -ldflags, -gcflags, static builds, cross-compilation, reproducible builds.
Security deep dive: input validation, authN/authZ, JWT/OAuth2, TLS/mTLS, secrets management, SBOM/dependency scanning.
High-performance architecture: caching, pooling, backpressure, async workflows, graceful degradation.
Cloud-native adaptation: Docker, Kubernetes basics, health probes, autoscaling, config/secrets, CI/CD, observability (logs/metrics/traces).
Best practices: code reviews, ADRs, release strategy, rollback, SLO/SLI fundamentals.
Capstone Project (High-Level Overview, 3–4 weeks)

Build an e-commerce backend with 4–6 Go microservices (Gateway, Catalog, Order, Payment, Notification).
Communication: synchronous REST + asynchronous Kafka events.
Persistence: one RDBMS service + one NoSQL-backed service.
Non-functional goals: secure APIs, tracing/metrics, performance target (e.g., p95 latency), load test report.
Delivery artifacts: architecture diagram, API specs, threat model, benchmark results, deployment manifests, demo walkthrough.
